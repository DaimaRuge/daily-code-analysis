## 整体架构设计

Agent Skills采用**Nx管理的 monorepo 架构**，清晰地分离了四个核心模块：

```
agent-skills/
├── packages/
│   ├── cli/              # 交互式命令行工具，用于技能安装、更新、删除
│   ├── mcp/              # MCP (Model Context Protocol) 服务器，为AI代理提供技能 catalog 访问
│   ├── marketplace/      # Next.js 构建的官方网站/市场前端
│   └── skills-catalog/   # 技能目录本体，包含所有经过验证的技能定义和注册表
├── libs/                 # 共享库（源码未完全展开）
├── tools/                # 构建/发布工具脚本
└── ...配置文件...
```

### 架构特点

1. **关注点分离**：四个包职责清晰，独立发布，可以单独使用
   - 用户可以只使用CLI工具安装技能
   - AI代理可以直接使用MCP服务器获取技能
   - 技能目录独立版本管理，方便更新

2. **安全优先设计**：
   - 所有技能经过Snyk安全扫描
   - 100%开源，无二进制闭源组件
   - CLI采用多重防御机制：路径隔离、符号链接防护、原子锁文件、审计日志
   - 内容哈希保证完整性

3. **多客户端支持**：
   - 支持主流AI编码助手（Claude Code、Cursor、Windsurf等）
   - 支持MCP协议接入
   - 支持全局安装和项目本地安装两种模式

## 核心模块分析

### 1. CLI 模块 (`packages/cli`)

#### 目录结构
```
src/
├── atoms/              # Jotai 状态原子（状态管理）
├── cli/                # 命令行操作实现
│   ├── audit.ts        # 审计日志
│   ├── cache.ts        # 缓存管理
│   ├── install.ts      # 安装逻辑
│   ├── remove.ts       # 删除逻辑
│   └── update.ts       # 更新逻辑
├── components/         # Ink 组件（基于React的交互式命令行UI）
├── hooks/              # React hooks
├── services/           # 业务服务
│   ├── config.ts       # 配置管理
│   ├── update-cache.ts # 缓存更新
│   ├── update-check.ts # 更新检查
│   └── ...
├── theme/              # 终端主题和颜色管理
├── utils/              # 工具函数
└── views/              # 不同向导页面视图
```

#### 核心技术栈
- **Ink**: React 驱动的交互式命令行UI框架
- **Jotai**: 轻量级状态管理
- **TypeScript**: 100%类型安全
- **Jest**: 单元测试，支持快照测试(PBT)

#### 关键设计点
1. **交互式向导**: 使用React+Ink实现了流畅的交互式安装体验，支持回退、搜索筛选、预览
2. **缓存机制**: 下载的技能缓存在`~/.cache/agent-skills/`，支持离线使用
3. **审计日志**: 记录所有操作，方便追溯和问题排查
4. **对称职安装**: 支持多种AI代理，一次选择多代理同时安装
5. **安全检查**: 安装前进行环境检查，避免路径穿越等问题

### 2. MCP 模块 (`packages/mcp`)

MCP是Model Context Protocol的缩写，这是一个标准化的AI代理工具调用协议。Agent Skills提供了MCP服务器，允许AI代理直接动态发现和加载技能。

#### 核心功能
```
src/tools/core/
├── list.ts     # 列出所有技能（按分类）
├── search.ts   # 按意图模糊搜索技能
├── skill.ts    # 读取技能的主要说明
└── fetcher.ts  # 获取技能的参考文件
```

#### 设计原则
- **渐进式披露**: 只在需要时获取技能内容，减少token消耗
- **模糊搜索**: 支持按用户意图搜索匹配的技能
- **缓存**: 和CLI共享缓存机制，避免重复下载

#### 暴露工具
| 工具 | 功能 |
|---|---|
| `list_skills` | 按分类浏览所有技能 |
| `search_skills` | 按意图模糊搜索 |
| `read_skill` | 加载技能主说明 |
| `fetch_skill_files` | 获取特定参考文件 |

### 3. Skills Catalog 模块 (`packages/skills-catalog`)

这是技能目录本体，所有经过验证的技能都存储在这里。

#### 技能结构
```
skills/
├── (category-name)/
│   └── skill-name/
│       ├── SKILL.md      # 主要技能说明（给AI阅读）
│       ├── templates/    # 文件模板（可选）
│       └── references/  # 参考文档（可选）
```

#### 注册表
- `skills-registry.json`: 包含所有技能的元数据，用于CLI和MCP快速检索，不需要克隆整个仓库就能获取目录信息
- 每次添加新技能都会自动更新注册表

#### 安全机制
- `security-scan-allowlist.yaml`: 允许的依赖列表，所有技能都经过Snyk扫描，确保没有已知漏洞

## 核心工作流程

### 用户安装技能流程
1. 用户执行 `npx @tech-leads-club/agent-skills`
2. CLI从CDN下载技能目录 (`skills-registry.json`)
3. 用户通过交互式向导选择技能、目标代理、安装方式
4. CLI从缓存/CDN下载技能文件
5. 校验完整性后安装到对应代理的配置目录
6. 记录操作到审计日志

### AI代理通过MCP使用技能流程
1. AI代理调用 `search_skills` 搜索匹配用户需求的技能
2. AI调用 `read_skill` 获取技能说明
3. AI根据技能说明指导用户或执行操作
4. 需要时调用 `fetch_skill_files` 获取参考文档

## 安全设计深度分析

Agent Skills最大的亮点就是安全设计，解决了当前AI技能市场的痛点：据报告，超过13%的市场技能包含严重漏洞。

### 多层防御体系

1. **入库前扫描**:
   - 每个PR自动使用Snyk Agent Scan扫描所有技能
   - 不满足安全要求的PR无法合并

2. **运行时防护**:
   - **路径清理**: 所有输入路径进行严格清理，防止路径穿越攻击
   - **符号链接检查**: 禁止安装过程创建符号链接，防止文件系统篡改
   - **原子写入**: 使用临时文件+原子重命名，避免部分写入导致的损坏
   - **锁文件**: 使用原子锁保证并发操作安全
   - **审计日志**: 所有操作都有日志记录，方便问题追溯

3. **分发安全**:
   - 所有技能内容可审计，100%文本文件，无二进制
   - 内容哈希校验，保证完整性
   - CDN分发，防止供应链篡改

## 优点与可改进点

### 优点
✅ **清晰的架构设计**: 模块化分离，职责清晰，易于维护  
✅ **安全优先**: 从流程到运行时多层防护，解决行业痛点  
✅ **优秀的用户体验**: 交互式CLI，支持键盘导航，分类搜索，多代理同时安装  
✅ **开放标准**: 支持MCP协议，方便各种AI代理接入  
✅ **完整的测试**: 核心逻辑都有单元测试，保证质量  

### 可改进点
🔍 **可以考虑**:
- 增加技能依赖管理：当前技能之间没有依赖管理，复杂技能组合可能需要用户手动处理
- 支持技能版本锁定：当前默认总是安装最新版，无法锁定特定版本
- 离线模式优化：虽然缓存已经实现，但可以进一步支持完全离线使用整个目录
