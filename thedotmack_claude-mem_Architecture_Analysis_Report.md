## 一、项目概述

### 1.1 核心定位

Claude-Mem 是 **Claude Code 的持久记忆插件**：

> "Persistent memory compression system built for Claude Code."

它自动捕捉 Claude Code 编程过程中的所有操作，并压缩存储到本地。Claude Code 下次运行时，可以查询历史记忆，不需要重复解释背景。

### 1.2 解决的问题

Claude Code 的记忆问题：
- **每次对话从零开始**：不记得之前做了什么
- **长对话 context 爆炸**：历史记录太长了
- **无法利用历史经验**：昨天的代码风格、偏好都不记得

### 1.3 Claude-Mem 的解法

```
编程会话 → 自动捕捉 → 压缩存储 → 下次可查询
```

## 三、核心功能

### 3.1 自动捕捉

Claude-Mem 自动捕捉：
- **文件修改历史**：每次改了什么文件，改了什么内容
- **命令执行记录**：运行的命令和输出
- **项目上下文**：项目结构、依赖、技术栈
- **对话摘要**：关键决策和结论

### 3.2 记忆压缩

| 技术 | 作用 |
|------|------|
| LZ4 压缩 | 快速压缩，节省存储 |
| 增量存储 | 只存新增，不重复 |
| 向量索引 | 支持语义搜索 |

### 3.3 查询召回

下次 Claude Code 启动时，可以问：
- "上次我们讨论的那个数据库方案是什么？"
- "这个项目的代码风格规范是什么？"
- "之前实现支付模块时遇到了什么问题？"

## 五、洞察总结

### 5.1 核心发现

1. **"自动捕捉"降低使用门槛**：不需要手动标记重要内容，Claude-Mem 自动处理。这让记忆变得真正无感。

2. **压缩+向量索引是聪明的工程选择**：用 LZ4 快速压缩存储，用向量索引支持语义搜索——既省空间，又能找到需要的内容。

3. **AGPL 许可证说明项目方的态度**：AGPL 是强开源许可证，保证派生项目也必须开源。

4. **27 种语言文档说明全球化做得好**：27 种语言的 README，说明项目在全世界都有用户。

5. **72k stars 是 Claude Code 插件中的高人气**：说明"记忆"是 Claude Code 用户的真实痛点。

### 5.2 局限性

- **仅限 Claude Code**：不适用于其他 AI 编程工具
- **AGPL 许可证**：商业使用有许可证要求
- **本地存储**：换机器需要同步记忆
