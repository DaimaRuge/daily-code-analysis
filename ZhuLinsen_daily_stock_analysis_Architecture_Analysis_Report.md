## 一、项目概述

### 1.1 核心定位

daily_stock_analysis 是一个**面向散户投资者的 AI 股票分析工具**，其核心理念是：用 AI 把专业分析师的工作自动化，让普通投资者每天早上能收到一份"决策仪表盘"，包含买卖建议、风险预警、操作检查清单。

项目支持多渠道推送（企业微信、飞书、Telegram、Discord、Slack、邮件），默认在每个工作日 18:00（北京时间）自动运行。用户也可以本地运行或 Docker 部署。

### 1.2 目标用户

- **有自选股但缺乏分析能力的散户**：不想每天盯盘，但希望有个 AI 帮他们做初步分析
- **需要每日报告推送的投资者**：希望在收盘后收到一份系统化的分析报告
- **有编程能力的量化爱好者**：可以本地部署、自定义策略、回测验证

### 1.3 关键指标

```
└─────────────────────────────────────────────────────────────────────┘
```

### 2.2 数据流设计

```
自选股配置 (STOCK_LIST)
    ↓
定时触发 / 手动触发
    ↓
多数据源并行获取（行情 + 新闻 + 舆情）
    ↓
AI 多维度分析（技术面 + 资金面 + 基本面 + 消息面）
    ↓
LLM 生成决策仪表盘
    ↓
多渠道推送（企业微信/飞书/Telegram等）
```

### 2.3 部署架构选择

| 部署方式 | 适用场景 | 成本 |
|---------|---------|------|
| **GitHub Actions（推荐）** | 零成本，5 分钟部署，无需服务器 | 免费 |
| **Docker** | 有服务器，想本地运行 | 服务器费用 |
| **本地定时任务** | 有 NAS 或本地机器 | 电力成本 |
| **FastAPI 服务** | 需要 API 接口供其他系统调用 | 服务器费用 |

GitHub Actions 是默认推荐的方案，因为它是零成本的，对于不想管理服务器的散户投资者来说非常友好。

### 3.4 决策仪表盘

核心输出是一份结构化的"决策仪表盘"：

```
🎯 2026-02-08 决策仪表盘
共分析3只股票 | 🟢买入:0 🟡观望:2 🔴卖出:1

📊 分析结果摘要
⚪ 中钨高新(000657): 观望 | 评分 65 | 看多
⚪ 永鼎股份(600105): 观望 | 评分 48 | 震荡
🟡 新莱应材(300260): 卖出 | 评分 35 | 看空

（单只股票详情包含：舆情情绪、业绩预期、风险警报、利好催化、最新动态）
```

这个仪表盘的设计目标是：**让散户投资者在 5 分钟内了解当日自选股的关键信息，并获得明确的操作建议**。

### 3.6 高级功能

#### AI 回测验证

对历史分析进行事后验证，查看方向准确率和模拟收益。这是一个有价值的功能——让投资者知道 AI 的分析在实际市场中表现如何。

#### Agent 问股

多轮策略问答，支持均线金叉/缠论/波浪等 11 种内置策略，支持 Web/Bot/API 全链路调用。

#### 大盘复盘

每日市场概览、指数表现、涨跌统计与板块强弱，支持 cn/hk/us/both 四种模式。

## 四、竞品对比

### 4.1 全局对比

| 工具 | 类型 | 目标用户 | 部署方式 | AI 集成 |
|------|------|---------|---------|---------|
| **daily_stock_analysis** | 每日分析 + 推送 | 散户投资者 | GitHub Actions / Docker / 本地 | ✅ 多模型支持 |
| **同花顺/东方财富** | 传统股票软件 | 散户投资者 | 桌面端 / App | ❌ |
| **聚宽/米筐** | 量化平台 | 量化投资者 | 云端研究 + 回测 | ✅ |
| **AutoGen/AgentScope** | Agent 框架 | 开发者 | 本地 | ✅ 但需自己开发 |
| **FinBERT 等金融模型** | 专业分析 | 专业分析师 | API | ✅ |

### 4.2 深度对比

#### vs 传统股票软件（同花顺、东方财富）

- **优势**：AI 驱动的决策建议，不是简单的数据展示；多渠道自动推送；免费开源
- **劣势**：没有实时行情（收盘后分析）；没有交易功能；数据源质量不如专业软件

#### vs 量化平台（聚宽、米筐）

- **定位不同**：量化平台面向专业量化投资者，强调回测和策略开发；daily_stock_analysis 面向普通散户，强调每日决策
- **技术门槛**：量化平台需要 Python 编程能力；daily_stock_analysis 零技术门槛（GitHub Actions 部署）

### 4.3 核心差异化价值

daily_stock_analysis 最大的价值在于**它把专业投研的工作流自动化了**，并通过 GitHub Actions 实现了零成本部署。

传统上，一个散户投资者想要获得类似的服务，需要：
1. 自己学会技术分析（看书、看教程）
2. 每天手动复盘（耗费大量时间）
3. 依赖券商软件（数据分散，分析靠人脑）

有了 daily_stock_analysis，你只需要配置好自选股和 API Key，每天就能收到一份 AI 生成的决策报告。

## 五、洞察总结

### 5.1 核心发现

1. **GitHub Actions 作为免费服务器的创新用法**：项目把 GitHub Actions 用作"每日定时分析的执行器"，这是一个非常聪明的设计——零成本、无服务器、可靠定时。对于不想付费买服务器的散户来说，这是最优解。

2. **多数据源聚合是核心能力**：项目支持 7 个行情数据源 + 6 个新闻搜索源 + 社交舆情，这种多源聚合能力是它与简单 stock bot 的本质区别。单一数据源总有失效的时候，多源备份保证了分析的可靠性。

3. **飞书推送是重要的国内渠道**：项目专门支持飞书 Webhook 作为推送渠道，这对于国内用户来说非常重要。相比 Telegram，飞书更适合国内用户的使用习惯。

4. **AIHubMix 的"一个 Key 用所有模型"是散户友好设计**：对于不想折腾多个 API 的散户来说，AIHubMix 提供了一个简单的解决方案——一个 Key，切换使用全系模型。

5. **决策仪表盘的可操作性是关键**：项目的核心输出不是"涨了还是跌了"这种信息，而是"买入/观望/卖出 + 评分 + 风险警报 + 操作清单"。这种结构化的、可操作的输出，是它区别于简单新闻聚合的关键。

### 5.2 局限性

- **时效性限制**：每日 18:00 推送，是收盘后的分析，不适合日内交易者
- **数据质量依赖**：行情数据主要来自免费数据源（如 AkShare），数据质量和完整性不如专业终端
- **非实时行情**：无法提供盘中实时分析，只能做收盘后的复盘和次日展望
- **API 额度限制**：GitHub Actions 环境下使用云端 API，有调用频率和成本限制

### 5.3 架构评分

| 维度 | 评分 | 说明 |
|------|------|------|
| **创新性** | ★★★★☆ | GitHub Actions 定时执行 + 多源聚合有创新 |
| **实用性** | ★★★★★ | 精准满足散户投资者的决策辅助需求 |
| **部署体验** | ★★★★★ | GitHub Actions 零成本，5 分钟完成部署 |
| **数据源覆盖** | ★★★★☆ | 覆盖全面，但免费数据源质量有限 |
| **输出质量** | ★★★★☆ | 决策仪表盘结构清晰，可操作性强 |

**综合评价**：daily_stock_analysis 是一个精准定位散户投资者需求的工具。它用 AI 把专业投研的工作流自动化，并通过 GitHub Actions 实现了零成本部署。对于不想付费买服务器、又希望每天获得股票分析报告的散户来说，这是目前最实用的开源解决方案。
