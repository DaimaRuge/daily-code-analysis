### 2.2 架构风格

该项目采用**文档驱动的去中心化架构**：

- **文档即数据**：所有技能信息存储在 Markdown 文件中
- **Git 版本控制**：利用 Git 进行变更跟踪和协作
- **GitHub 工作流**：通过 PR、Issue、Discussion 进行社区协作
- **扁平文件结构**：无复杂的数据库或后端服务

### 2.3 关键设计决策

1. **Markdown 作为主要数据格式**
   - 优点：人类可读、易于编辑、Git 友好
   - 权衡：需要额外的解析工具进行结构化访问

2. **按功能领域分类**
   - 优点：便于用户按需求查找
   - 权衡：有些技能可能跨多个分类

3. **GitHub PR 驱动的贡献流程**
   - 优点：利用成熟的代码审查机制
   - 权衡：对新贡献者有一定学习曲线
