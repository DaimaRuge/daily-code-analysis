## 一、项目概述

### 1.1 核心定位

Edit-Banana 是一个**图片到可编辑图表的转换工具**。它的核心能力是：**上传一张图片（流程图、架构图、技术示意图、公式等），AI 自动识别图中的所有元素，输出一个完整的 DrawIO (XML) 文件，你可以在 DrawIO 中自由编辑**。

一句话：**"截图里的图，变成你能拖拽的图。"**

### 1.2 解决的问题

这个问题的痛点非常普遍：
1. 你在文档/论文/PPT里看到一个好图
2. 你想修改它（改文字、换颜色、调整布局）
3. 但它是图片——不可编辑
4. 你只能重新画——花大量时间

Edit-Banana 的解法：**用 AI 识别图片中的图表元素，还原为原生可编辑格式。**

### 1.3 目标用户

- **开发者**：看到好的架构图想复用
- **研究人员**：论文中的图需要调整
- **产品经理**：流程图需要修改
- **PPT 制作者**：从别人那里拿到截图想改
- **学生**：课程中的概念图想重新整理

### 1.4 关键指标

| 指标 | 数值 |

| GitHub Stars | ⭐5k+ |

### 2.2 核心流水线详解

**Step 1: SAM 3 分割**

Edit-Banana 使用**经过微调的 SAM 3** 来分割图片中的图表元素。SAM（Segment Anything Model）是 Meta 的通用分割模型，Edit-Banana 对其 Mask Decoder 进行了微调，使其更适合图表场景：

- 识别形状（矩形、圆形、菱形等）
- 识别连接线（箭头、直线、曲线）
- 识别文本框和标签
- 保持元素的层次关系

**Step 2: 并行文本提取**

文本提取是双管齐下的：

| 组件 | 用途 | 特点 |
|------|------|------|
| Tesseract OCR | 普通文本检测 | 本地运行，不依赖网络 |
| Pix2Text | 数学公式识别 | 支持 LaTeX 转换 |
| Crop-Guided Strategy | 高分辨率裁剪 | 对文本/公式区域做高分辨率裁剪后送入识别引擎 |

**Step 3: DrawIO XML 生成**

关键创新在这里——不是简单地把识别结果拼在一起，而是：
- 保留原始图表的**布局逻辑**（元素之间的相对位置）
- 1:1 还原**颜色匹配**（填充色、边框色）
- 还原**箭头样式**（虚线、实线、箭头形状）
- 还原**边框样式**（线宽、虚线）
- 所有元素**独立可选**（支持后续编辑）

### 2.3 用户系统架构

## 三、技术亮点分析

### 3.1 SAM 3 微调

Edit-Banana 对 SAM 3 的 Mask Decoder 做了针对性微调。普通的 SAM 3 对自然图像分割很好，但对图表元素（规则形状、连接线等）可能不够精确。微调后的版本在图表场景有更好的表现。

### 3.2 Crop-Guided 文本提取

这是 Edit-Banana 的一个重要工程优化：
1. SAM 3 首先识别出文本区域（粗略定位）
2. 对这些区域做高分辨率裁剪（不压缩）
3. 将裁剪后的高清图像送入 OCR/公式引擎
4. 获得更准确的文本识别结果

### 3.3 全流程高保真

对比演示的 4 个场景：
- **流程图**：布局逻辑保持
- **多层架构图**：层次结构保持
- **技术原理图**：连接关系保持
- **科学公式**：LaTeX 转换

### 4.1 全局对比

| 工具 | 定位 | 技术路线 | 输出格式 | 开源 | 部署 |
|------|------|---------|---------|------|------|

### 4.2 深度对比

#### vs Mathpix

| 维度 | Edit-Banana | Mathpix |
|------|-------------|---------|
| **定位** | 完整图表还原 | 文本/公式识别 |
| **输出** | DrawIO XML（完整可编辑图表） | LaTeX/Markdown |
| **元素还原** | 形状+颜色+箭头+文字 | 仅文字 |
| **编辑性** | 在 DrawIO 中直接拖拽 | 文本调整 |
| **开源** | ✅ | ❌ |
| **价格** | 免费额度 + 按次计费 | 订阅制 |

#### vs GPT-4o 直接识别

GPT-4o 也可以"看图说话"，但问题在于：
- 输出是文本描述，不是可编辑的 XML
- 精度不如专门的流水线（分割 + OCR）
- 无法精确保持布局和样式

### 4.3 核心差异化价值

1. **端到端的图表还原**：不是 OCR + 文本描述，而是完整的可编辑图表
2. **DrawIO XML 输出**：draw.io 是免费且广泛使用的图表工具
3. **SAM 3 微调**：专为图表场景优化
4. **高保真**：颜色、样式、布局的 1:1 还原
5. **在线体验**：开箱即用

## 五、洞察总结

### 5.1 核心发现

1. **这是一个"小而美"的 AI 工具**：Edit-Banana 解决了一个具体但高频的问题——图片图表的可编辑化。它不是 AGI，但非常有实用价值。

2. **SAM 3 + OCR 的组合是经典的 AI 工程**：没有使用一个端到端大模型，而是用 SAM 3 做分割、Tesseract 做 OCR、Pix2Text 做公式识别，最后用自定义逻辑融合结果。这种"组合式 AI"架构成熟且可控。

3. **在线版的付费模式很合理**：新用户 10 次免费试用，之后按次付费。这既防止了滥用，又给了用户充分的体验机会。

4. **GitHub 仓库落后于在线服务**：README 明确说"GitHub 仓库目前落后于我们的在线服务"，建议用户使用在线版本。这意味着开源版本主要是展示和引流，核心功能在云端。

5. **5k stars 反映了真实需求**：虽然不是爆款项目，但 stable 的增长说明这是一个真实有用的工具。

### 5.2 局限性

- **仅支持 DrawIO 输出**：不支持其他图表格式（Visio、LucidChart 等）
- **仅处理图表**：不适合自然场景照片
- **中文支持**：Tesseract OCR 的中文识别效果可能不如专门的中文 OCR 引擎
- **本地部署 vs 在线体验**：本地部署的版本可能功能不完整

### 5.3 架构评分

| 维度 | 评分 | 说明 |
|------|------|------|
| **创新性** | ★★★☆☆ | 组合已有 AI 能力，工程创新为主 |
| **实用性** | ★★★★★ | 解决真实高频需求 |
| **技术深度** | ★★★☆☆ | SAM 微调 + OCR 组合，技术栈经典 |
| **用户体验** | ★★★★★ | 在线版开箱即用 |
| **商业潜力** | ★★★★☆ | 按次付费模式可行 |

**综合评价**：Edit-Banana 是一个务实、实用的 AI 工具。它没有追逐 AGI，而是专注解决"图片图表无法编辑"这个具体痛点。通过 SAM 3 + OCR + DrawIO 的组合架构，让静态图片变成可自由编辑的原生图表。对于经常需要从文档/论文/截图中复用图表的用户来说，这是一个非常实用的工具。
