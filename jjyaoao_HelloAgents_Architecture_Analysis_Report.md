## 一、项目概述

### 1.1 核心定位

HelloAgents 是一个**基于 OpenAI 原生 API 构建的生产级多智能体框架**：

> "🤖 生产级多智能体框架 - 工具响应协议、上下文工程、会话持久化、子代理机制等 16 项核心能力"

### 1.2 16 项核心能力

| 能力 | 说明 |
|------|------|
| **ToolResponse** | 工具响应协议 |
| **HistoryManager** | 上下文工程 |
| **TokenCounter** | Token 计数 |
| **SessionStore** | 会话持久化 |
| **TaskTool** | 子代理机制 |
| **乐观锁** | 文件编辑的乐观锁 |
| **CircuitBreaker** | 熔断器 |
| **Skills** | 知识外化 |
| **TodoWrite** | 进度管理 |
| **DevLog** | 决策记录 |
| **SSE** | 流式输出 |
| **异步生命周期** | 异步架构 |
| **TraceLogger** | 可观测性 |
| **日志系统** | 四种日志范式 |
| **LLM/Agent 基类** | 灵活重构 |

### 1.3 目标用户

- **想学习 Agent 开发**：有学习版本分支，配 Datawhale 教程
- **需要生产级框架**：16 项核心能力覆盖工程化需求
- **多 LLM 提供商**：支持 OpenAI / Claude / Gemini / 本地推理

### 2.2 LLM 提供商支持

| 类型 | 示例 |
|------|------|
| **云端 API** | OpenAI, DeepSeek, Qwen, Kimi, 智谱 GLM |
| **本地推理** | vLLM, Ollama, SGLang |
| **Claude** | Anthropic 官方 |
| **Gemini** | Google 官方 |

框架根据 `base_url` 自动选择适配器。

## 五、洞察总结

### 5.1 核心发现

1. **16 项核心能力是诚意之作**：不是简单封装，而是有 CircuitBreaker、乐观锁、熔断器等工程化能力。

2. **Datawhale 教程是差异化优势**：配套的学习版本 + 完整教程，让初学者可以平滑上手。

3. **三适配器架构**：OpenAI / Anthropic / Gemini 根据 URL 自动切换，用户不需要关心细节。

4. **会话持久化 + 可观测性**：这些都是生产级应用需要的，HelloAgents 都有。
